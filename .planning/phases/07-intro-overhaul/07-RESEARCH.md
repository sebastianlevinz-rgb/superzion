# Phase 7: Intro Overhaul - Research

**Researched:** 2026-03-19
**Domain:** Phaser 3 cinematic intro scene -- flag animations, title screen, procedural SFX, camera shake
**Confidence:** HIGH

## Summary

Phase 7 enhances the existing `GameIntroScene.js` -- a 25-second three-act cinematic intro that already has boss sprites, missiles, explosions, jets, tanks, Iron Dome, and camera shake. The infrastructure is solid. The four requirements each address a specific gap: (1) flags are generated in `ParadeTextures.js` but never displayed in the intro, (2) the title screen uses monospace font rather than the specified giant Maguen David + arcade font layout, (3) SFX are currently all `playExplosion()` with no differentiation for missiles/jets/gunfire, and (4) camera shake already exists but needs to be verified as consistently applied to every explosion event.

The `ParadeTextures.js` already exports `createIranFlag`, `createLebanonFlag`, `createPalestineFlag`, and `createIsraelFlag` -- all using the `wavingSheet()` helper that generates 4-frame animated spritesheets with a `_wave` animation key. These are called in `generateAllParadeTextures(scene)` which runs at the top of `GameIntroScene.create()`. The flags just need to be instantiated as sprites and played with their wave animation in Act 1.

The title screen (Act 3, ~5s mark) currently shows "SUPERZION" at 64px monospace with gold shadow. INTRO-02 requires: (a) a giant golden semi-transparent Maguen David behind the hero, (b) "SUPERZION" in wide thick arcade-style font, and (c) a larger subtitle below. The `starOfDavid()` helper already exists in ParadeTextures.js and can be repurposed, or a large Maguen David can be drawn via Phaser graphics directly in Act 3.

IntroMusic.js already provides a full 25-second psytrance score at 145 BPM with synchronized impact booms, acid squelches, and filter sweeps. SoundManager.js has `playExplosion()`, `playMissilePass()`, `playMissileWarning()`, `playJetEngine()`, `playAfterburner()`, `playBombDrop()`, and `playBombImpact()` -- but the intro currently only calls `playExplosion()` for everything. INTRO-03 requires adding distinct SFX calls (missile whoosh, varied booms, jet Doppler flyby, gunfire). IntroMusic.js also exports standalone SFX functions: `playMissileWhoosh()`, `playJetFlyby()`, `playTankRumble()`, `playMarchSteps()`, `playWindAmbient()` -- these are already written but never called from the intro scene.

**Primary recommendation:** This is a wiring and enhancement phase, not an architecture phase. The flag textures, SFX functions, and camera shake API all exist already. The work is: instantiate flags in Act 1, restyle Act 3 title screen, replace generic `playExplosion()` calls with specific SFX, and audit/add camera shake to all explosion callbacks.

**What:** Use `this.cameras.main.shake(duration, intensity)` consistently
**When to use:** Every `_spawnExplosion()` call and every delayed explosion event
**Example:**
```javascript

// duration_ms: 100-400 for intro explosions
// intensity: 0.005 (subtle) to 0.03 (heavy, title impact)
```

## Common Pitfalls

### Pitfall 1: Flag Sprite Scale
**What goes wrong:** Flags render at their canvas size (140x104) which may be too large or too small for the intro scene context
**Why it happens:** `wavingSheet()` generates at 140x90 base size + 14px padding = 140x104
**How to avoid:** Use `.setScale()` to fit flags proportionally. At scale 0.5 they are ~70x52px, suitable for parade context. Test visually.
**Warning signs:** Flags overlapping bosses or each other, or appearing as tiny dots

### Pitfall 2: SFX Volume Stacking
**What goes wrong:** Multiple SFX playing simultaneously creates a loud distorted mess
**Why it happens:** Each SFX function creates its own gain node connected to masterGain/musicGain. Multiple concurrent sounds sum their amplitudes.
**How to avoid:** Keep individual SFX volumes low (0.08-0.15). The IntroMusic exported functions already have reasonable defaults. Don't double-fire SFX (e.g., both `playExplosion()` AND `_impactBoom()` at the same timestamp).
**Warning signs:** Audio clipping, distorted peaks on explosions

### Pitfall 3: Font Availability for Arcade Text
**What goes wrong:** Specifying 'Impact' or 'Arial Black' may not render on all systems, falling back to serif or sans-serif
**Why it happens:** Browser font availability varies by OS
**How to avoid:** Use a CSS font stack: `'"Impact", "Arial Black", "Trebuchet MS", sans-serif'`. Impact is available on Windows/Mac. For truly arcade-style, consider Phaser bitmap text, but the project has never used it. Alternatively, use CSS `letter-spacing` with bold monospace for a close approximation. The key visual requirement is "wide thick" -- `fontWeight: 'bold'` combined with `fontSize: '72px'` and Phaser's text shadow blur will achieve the arcade look even with fallback fonts.
**Warning signs:** Title text appears thin or narrow compared to expectation

### Pitfall 4: Act Cleanup Race Conditions
**What goes wrong:** `_clearAct()` destroys flag sprites while their animations are still playing, causing errors
**Why it happens:** `actObjects.push(sprite)` followed by `_clearAct()` calls `.destroy()` on everything, which is correct behavior, but if any tween/timer references a destroyed object after clear, it throws
**How to avoid:** Always guard callbacks with `if (this.skipped) return;` -- this pattern is already used throughout the scene. Ensure flag sprites and their animations are added to `actObjects` so they get cleaned up properly.
**Warning signs:** Console errors about destroyed game objects after skipping

### Pitfall 5: Duplicate Camera Shake Stacking
**What goes wrong:** Multiple simultaneous `camera.shake()` calls cancel each other out or create jittery artifacts
**Why it happens:** Phaser 3's camera shake replaces the current shake when a new one starts (it does not stack)
**How to avoid:** Don't call shake on every single explosion if they happen within 100ms of each other. The existing pattern of calling shake at specific timed intervals (boss entries, beat-synced) is correct. For `_spawnExplosion()`, only add shake if not already shaking (or use a minimum delay between shakes).
**Warning signs:** Shakes feel inconsistent or some explosions don't shake at all because a previous shake overwrites

### Pitfall 6: IntroMusic SFX Audio Context Timing
**What goes wrong:** SFX functions from IntroMusic.js require `ctx`, `gainNode`, and `startTime` parameters, different from SoundManager's parameter-free pattern
**Why it happens:** IntroMusic exports standalone functions with explicit audio routing, while SoundManager methods are self-contained singletons
**How to avoid:** When calling IntroMusic SFX functions, always pass `mm.ctx`, `mm.musicGain` (or `this._introMusic._gainNode`), and `mm.ctx.currentTime`. Track returned nodes for cleanup in `_nodes` array.
**Warning signs:** SFX don't play at all, or play through wrong gain node (not affected by mute toggle)

### SFX Diversification
```javascript
// Add to SoundManager.js:
playGunfire(burstCount = 3) {
  this._init();
  if (!this.ctx || !this.masterGain) return;
  const t = this.ctx.currentTime;
  for (let i = 0; i < burstCount; i++) {
    const buf = this._createNoiseBuffer(0.05);
    const src = this.ctx.createBufferSource();
    src.buffer = buf;
    const hp = this.ctx.createBiquadFilter();
    hp.type = 'highpass';
    hp.frequency.value = 2000;
    const gain = this.ctx.createGain();
    gain.gain.setValueAtTime(0.15, t + i * 0.08);
    gain.gain.exponentialRampToValueAtTime(0.001, t + i * 0.08 + 0.04);
    src.connect(hp);
    hp.connect(gain);
    gain.connect(this.masterGain);
    src.start(t + i * 0.08);
    src.stop(t + i * 0.08 + 0.05);
  }
}

// In GameIntroScene, wire specific SFX to events:
// Missile launch:
import { playMissileWhoosh } from '../systems/IntroMusic.js';
// In _spawnMissile():
const mm = MusicManager.get();
if (mm.ctx && mm.musicGain) playMissileWhoosh(mm.ctx, mm.musicGain, mm.ctx.currentTime);

// Jet strike:
import { playJetFlyby } from '../systems/IntroMusic.js';
// In _spawnJetStrike():
if (mm.ctx && mm.musicGain) playJetFlyby(mm.ctx, mm.musicGain, mm.ctx.currentTime);

// Gunfire on hero pistol shot:
SoundManager.get().playGunfire(1);
```

## Open Questions

1. **Flag placement context: with bosses or separate?**
   - What we know: Bosses appear at x=150, W/2-100, W/2+100, W-150. Flags exist for Iran, Lebanon, Palestine, Israel.
   - What's unclear: Should flags appear behind each boss (enemy flags) plus Israel separately, or should all 4 appear in a different arrangement?
   - Recommendation: Enemy flags (Iran, Lebanon, Palestine) appear behind their respective bosses in Act 1. Israel flag appears in Act 2 or Act 3 with the hero. This maps naturally to the narrative.

2. **Exact arcade font appearance**
   - What we know: Requirement says "wide thick arcade-style font"
   - What's unclear: Whether the user expects a custom bitmap font or CSS-available bold fonts
   - Recommendation: Use Impact/Arial Black with letter-spacing. If visual result is insufficient, fall back to drawing text on a canvas with custom stroke for outline effect. The project has never used bitmap fonts.

3. **Gunfire SFX does not exist in SoundManager**
   - What we know: No `playGunfire()` method exists. The hero fires a "pistol" in Act 3 at 4s.
   - What's unclear: Whether gunfire is needed only for the hero shot or also during Act 1/2
   - Recommendation: Add a simple `playGunfire()` to SoundManager. Wire it to the hero's pistol shot in Act 3, and optionally to some Act 1 boss entries for variety.
