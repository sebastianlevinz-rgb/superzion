Create `superzion/src/scenes/PlatformerScene.js` as a complete Phaser Scene. This is the main deliverable of Phase 32.

**File structure:**

```
// Header with ASCII art dividers (project convention)
import Phaser from 'phaser'
import { createPlatformerLevel1Textures } from '../utils/PlatformerLevel1Textures.js'
import SoundManager from '../systems/SoundManager.js'
import MusicManager from '../systems/MusicManager.js'
import { showControlsOverlay } from '../ui/ControlsOverlay.js'

const W = 960, H = 540
const WORLD_WIDTH = 4800  // ~5 screens wide
const PLAYER_SPEED = 200
const JUMP_VELOCITY = -420
const COYOTE_TIME = 80  // ms
const JUMP_BUFFER = 100 // ms

export default class PlatformerScene extends Phaser.Scene {
  constructor() { super('PlatformerScene') }
  create() { ... }
  update(time, delta) { ... }
  // Private methods...
}
```

**create() method -- setup everything:**

1. **Background color**: `this.cameras.main.setBackgroundColor('#0a0a1a')` (deep night)
2. **Music**: `MusicManager.get().playLevel1Music()` (same music as Bomberman phase -- continuity)
3. **Textures**: `createPlatformerLevel1Textures(this)`
4. **Controls overlay**: `showControlsOverlay(this, 'ARROWS: Move/Jump | ESC: Pause')`
5. **Physics world bounds**: `this.physics.world.setBounds(0, 0, WORLD_WIDTH, H)`
6. **Camera bounds**: `this.cameras.main.setBounds(0, 0, WORLD_WIDTH, H)`
7. Call private setup methods: `_createBackground()`, `_createPlatforms()`, `_createPlayer()`, `_createObstacles()`, `_createGuards()`, `_createTarget()`
8. **Camera follow**: `this.cameras.main.startFollow(this.player, true, 0.1, 0.1)`
9. **Fade in**: `this.cameras.main.fadeIn(500, 0, 0, 0)`
10. **Input keys**: cursors via `this.input.keyboard.createCursorKeys()`, plus ESC for pause
11. **HP system**: `this.hp = 3`, `this.maxHp = 3`, `this.invulnerable = false`, `this.invulnTimer = 0`
12. **HUD**: Simple HP display using `this.add.text()` with `setScrollFactor(0)` showing hearts or HP count
13. **Game state flags**: `this.gameOver = false`, `this.transitioning = false`

**_createBackground() -- 4-layer parallax:**

Use TileSprite for each layer, covering full world width:
- Layer 0 (depth -10): `plt_stars_sky` -- stars/moon, scroll factor 0.02 (barely moves)
- Layer 1 (depth -8): `plt_mountains` -- Alborz range, scroll factor 0.1
- Layer 2 (depth -6): `plt_skyline` -- Tehran skyline with towers, scroll factor 0.3
- Layer 3 (depth -4): `plt_near_buildings` -- close buildings, scroll factor 0.6

Each TileSprite: `this.add.tileSprite(WORLD_WIDTH/2, yPos, WORLD_WIDTH, height, textureKey).setScrollFactor(scrollFactor).setDepth(depth)`

IMPORTANT: TileSprites with scroll factors handle parallax automatically when camera moves. No manual tilePositionX update needed -- Phaser's camera system + scrollFactor does this.

**_createPlatforms() -- 10-12 rooftop platforms:**

Create a `this.platforms = this.physics.add.staticGroup()`.

Platform layout (left to right, generally ascending with gaps):
```
{ x: 200, y: 420, key: 'plt_roof_flat', scaleX: 3 }      // Starting rooftop (wide)
{ x: 650, y: 390, key: 'plt_roof_balcony' }                // First jump
{ x: 950, y: 360, key: 'plt_roof_flat', scaleX: 1.5 }     // Building
{ x: 1300, y: 330, key: 'plt_roof_dome' }                  // Mosque dome
{ x: 1650, y: 370, key: 'plt_roof_flat', scaleX: 2.5 }    // Long building (guard here)
{ x: 2050, y: 340, key: 'plt_roof_cornice' }               // Narrow ledge
{ x: 2350, y: 310, key: 'plt_roof_balcony' }               // Higher balcony
{ x: 2650, y: 350, key: 'plt_roof_flat', scaleX: 2 }      // Building (guard here)
{ x: 3050, y: 320, key: 'plt_roof_cornice' }               // Narrow approach
{ x: 3350, y: 290, key: 'plt_roof_flat', scaleX: 1.5 }    // High building
{ x: 3700, y: 350, key: 'plt_roof_flat', scaleX: 2 }      // Pre-target
{ x: 4200, y: 380, key: 'plt_roof_flat', scaleX: 3 }      // Target building roof (wide)
```

For each platform: `const p = this.platforms.create(x, y, key)`. If scaleX specified, `p.setScale(scaleX, 1).refreshBody()` to update physics body to match visual.

Below the lowest platform, add an invisible kill zone: `this.killZone = this.add.zone(WORLD_WIDTH/2, H + 20, WORLD_WIDTH, 40)` with `this.physics.add.existing(this.killZone, true)` and overlap detection to kill player if they fall off.

**_createPlayer() -- platformer player character:**

```javascript
this.player = this.physics.add.sprite(200, 350, 'plt_player_idle')
this.player.setCollideWorldBounds(true)
this.player.body.setSize(28, 52)       // Slightly smaller than 64x64 for better collision feel
this.player.body.setOffset(18, 12)     // Center the collision box, feet at bottom
this.player.setDepth(10)
this.player.setScale(0.8)              // 64px sprite at 0.8 = ~51px -- good for the world scale
```

Add collider: `this.physics.add.collider(this.player, this.platforms)`

Player state: `this.lastOnGround = 0` (for coyote time), `this.jumpBufferTimer = 0`

Animation state tracking: `this.playerAnimState = 'idle'`, `this.playerRunFrame = 0`, `this.playerRunTimer = 0`

**_createObstacles() -- cameras, searchlights, electric wire:**

Create arrays to track: `this.cameras_ = []`, `this.searchlights = []`, `this.wires = []`

Place obstacles on/near specific platforms:
- **Security cameras** (3-4): Mount on buildings between platforms. Each has a position, sweep angle range, sweep speed. Vision cone drawn as a semi-transparent triangle in update(). Contact with cone = damage.
  - Camera at x=900 y=320 (sweeps left-right across platform gap)
  - Camera at x=1900 y=300 (watches over long building)
  - Camera at x=3200 y=260 (near target approach)

- **Searchlights** (2): Ground-mounted spotlight that sweeps. Similar to camera but larger cone, slower sweep.
  - Searchlight at x=1500 y=370 (sweeps across dome area)
  - Searchlight at x=2900 y=350 (before final approach)

- **Electric wires** (2-3): Horizontal hazard lines between buildings that flash on/off.
  - Wire at x=1150 y=345 width=100 (between platforms, timed 1s on / 1s off)
  - Wire at x=2500 y=325 width=80 (higher up, timed 0.8s on / 1.2s off)
  - Wire at x=3500 y=300 width=100 (near target)

For each obstacle type:
- Draw vision cones / active zones using `this.add.graphics()` in update loop
- Check overlap with player position (point-in-triangle for cones, rect overlap for wires)
- On contact: call `_damagePlayer()`

**_createGuards() -- patrolling guards on platforms:**

Create 3 guards on the wider platforms:
```javascript
this.guards = []
// Guard 1: patrols the long building at y=370
this.guards.push(new RooftopGuard(this, 1600, 338, 100))
// Guard 2: patrols the building at y=350
this.guards.push(new RooftopGuard(this, 2600, 318, 80))
// Guard 3: patrols pre-target area
this.guards.push(new RooftopGuard(this, 4100, 348, 120))
```

**RooftopGuard class** (inline in PlatformerScene.js or at bottom of file):
- Constructor: `scene.physics.add.sprite(x, y, 'plt_guard_walk_0')`
- `body.setSize(24, 52)`, `body.setOffset(20, 12)` -- feet aligned with sprite bottom
- CRITICAL: `scene.physics.add.collider(guard.sprite, scene.platforms)` -- guards must collide with platforms
- Guards use gravity (default 900) -- they fall onto platforms like the player
- Patrol: walk left/right within `patrolMinX`/`patrolMaxX`, reverse direction at bounds
- Walk animation: cycle through `plt_guard_walk_0..3` every 150ms
- ONLY move when `body.blocked.down` is true (on ground) -- NEVER float
- `console.log` verification: Log guard Y position vs platform Y on spawn to verify feet touch ground (LVL1-05)
- Contact damage: `scene.physics.add.overlap(scene.player, guard.sprite, () => scene._damagePlayer())`

**_createTarget() -- target building at end of level:**

Place target building visual at x=4400, using `plt_target_building` texture.
Create an overlap trigger zone at the open window position:
```javascript
this.targetZone = this.add.zone(4400, 280, 60, 60) // window area
this.physics.add.existing(this.targetZone, true)
this.physics.add.overlap(this.player, this.targetZone, () => this._enterBuilding())
```

Add a subtle golden glow/pulse around the window area using a graphics object or tween on a rectangle.

**_enterBuilding() -- transition to Bomberman (LVL1-06):**

```javascript
_enterBuilding() {
  if (this.transitioning) return
  this.transitioning = true
  this.player.body.setVelocity(0, 0)
  this.player.body.setAllowGravity(false)

  // Visual: zoom into window + fade
  this.cameras.main.zoomTo(2.5, 800, 'Power2')
  this.cameras.main.fadeOut(800, 0, 0, 0)

  // Sound: dramatic entry
  SoundManager.get().playExplosion()

  this.time.delayedCall(1000, () => {
    this.scene.start('GameScene')
  })
}
```

**update(time, delta) -- main loop:**

1. If `this.gameOver || this.transitioning` return early
2. **Player input handling:**
   - Left/right arrows: `this.player.body.setVelocityX(dir * PLAYER_SPEED)`, flip sprite accordingly
   - No input: `this.player.body.setVelocityX(0)`
   - Jump (up arrow): Only if on ground OR within coyote time. Apply `body.setVelocityY(JUMP_VELOCITY)`. Play `SoundManager.get().playJump()`.
   - Track `lastOnGround` for coyote time: if `body.blocked.down`, set `lastOnGround = time`
   - Jump buffer: if up pressed while airborne, set `jumpBufferTimer = time`. On next ground contact, auto-jump if within JUMP_BUFFER window.
3. **Player animation:**
   - On ground + moving: cycle `plt_player_run_0..3` every 120ms
   - On ground + still: `plt_player_idle`
   - In air: `plt_player_jump`
4. **Guard updates:** `for (const g of this.guards) g.update(delta)`
5. **Obstacle updates:**
   - Camera/searchlight sweep: update angle, redraw vision cone graphics, check player overlap
   - Electric wire: update timer, toggle visibility, check overlap when active
6. **Invulnerability timer:** Decrement, clear tint when done
7. **Kill zone check:** If player.y > H, call `_playerDeath()`
8. **HUD update:** Refresh HP display

**_damagePlayer():**
- If `this.invulnerable || this.hp <= 0` return
- `this.hp--`
- `this.invulnerable = true`, `this.invulnTimer = 1500` (1.5s)
- Player flash: `this.player.setTint(0xff0000)`, then clear after 100ms
- Camera shake: `this.cameras.main.shake(200, 0.005)`
- `SoundManager.get().playHit()`
- If `this.hp <= 0`: `_playerDeath()`

**_playerDeath():**
- `this.gameOver = true`
- Player falls off screen or plays death animation
- After 1.5s delay: `this.scene.restart()` (restart platformer)
- Show brief "MISSION FAILED" text

**Pause system:**
- ESC key toggles pause overlay (dark semi-transparent rectangle + "PAUSED" text + "R: Restart | Q: Quit to Menu")
- During pause: `this.physics.pause()` / `this.physics.resume()`

**IMPORTANT implementation notes:**
- Do NOT change global gravity -- it's 900 from main.js, which is perfect for a platformer
- GameScene sets its own gravity to 0 in its create() -- no conflict
- All obstacle vision cone graphics must be cleared and redrawn each frame (use `graphics.clear()`)
- Guard console.log verification per LVL1-05: On guard creation, log `Guard spawned at y=${y}, will land on platform at y=${platformY}`
- Player sprite depth = 10, guards depth = 9, obstacles depth = 8, platforms depth = 5, background depth < 0
